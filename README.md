### **Описание backend-части проекта**

Backend написан на **Spring Boot (Java)** и предоставляет API для управления светодиодной лентой через **ESP32**.

---

### **Компоненты backend**

1. **`Led` (Модель)**
    - Пока пустая, но может содержать:
        - Состояние светодиода (вкл/выкл).
        - Текущий режим (радуга, мигание, громкость).
        - Цвет (если поддерживается RGB).

2. **`LedController` (Контроллер)**
    - Принимает HTTP-запросы от фронтенда.
    - Доступные эндпоинты:
        - **`/rainbow`** – включает режим "Радуга".
        - **`/blink`** – включает режим "Мигание".
        - **`/volume`** – изменяет режим в зависимости от громкости (если подключен микрофон).

3. **`Esp32Service` (Сервис)**
    - Отправляет команды на **ESP32** через HTTP (например, `GET http://esp32-ip/rainbow`).
    - Обрабатывает ответ от микроконтроллера (успех/ошибка).

4. **`RestTemplate` (HTTP-клиент)**
    - Используется в `Esp32Service` для взаимодействия с ESP32.

---

### **Как это работает?**
1. Фронтенд отправляет запрос (например, `POST /api/led/rainbow`).
2. `LedController` принимает его и вызывает `Esp32Service`.
3. `Esp32Service` отправляет команду на **ESP32**, который управляет светодиодами.
4. Если нужно сохранять состояние, можно добавить **`LedRepository` (JPA/H2/PostgreSQL)**.
